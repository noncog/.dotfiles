* Old Info:
My system configuration files are managed with a git --bare repository.

This allows easy versioning and installation of dotfiles without symlinks or any form of moving/copying.

Simply put, the files are cloned to where they belong!

- To use these dotfiles as is, follow the usage section.
- To setup your own git --bare managed dotfiles follow the short setup guide below!
* Usage
If you're on a debian-based distribution, you can use my [[https://github.com/noncog/debian-autosetup][debian-autosetup]] script to automatically install my system setup, software, and dotfiles. Or easily fork/modify it to install your own software and dotfiles!

Otherwise, _to just install dotfiles_ (like these)  managed with a bare repository on a fresh system _see the cloning section below_!
* Short Setup Guide
This is the absolute bare minimum guide one could use to start managing their dotfiles with a bare repository. For more information, there are numerous guides on the internet.

A word of advice:
- See Recommendations section below for important GitHub practices before beginning.
- You do not need a .gitignore, so long as you never add .dotfiles to your .dotfiles. 
** Initial Setup
Create a bare repository and an alias command to control it. Then hide all untracked files.
#+BEGIN_SRC
git init --bare $HOME/.dotfiles
echo "alias dotfiles='/usr/bin/git --git-dir=$HOME/.dotfiles/ --work-tree=$HOME'" >> $HOME/.bashrc
dotfiles config --local status.showUntrackedFiles no
#+END_SRC
** Adding Files
#+BEGIN_SRC
dotfiles add .bashrc
dotfiles commit -m "Added .bashrc"
#+END_SRC
** First Push / Remote Setup
#+BEGIN_SRC
dotfiles remote add origin <your dotfiles repo>
dotfiles push -u origin master
#+END_SRC
** Commands
After everything is setup, you can use all git commands with the dotfiles alias.
#+BEGIN_SRC
# common commands
dotfiles push
dotfiles status
dotfiles log
#+END_SRC
** Recommendations For GitHub Users
*Use the private noreply email* provided by GitHub to prevent leaking your personal email in git commits.
- Provided by GitHub [[https://github.com/settings/email][here]].
#+BEGIN_SRC
- set on the command line using:
git config --global user.email <noreply-email>
- check it was set using:
git config --global user.email
#+END_SRC
*Use personal access tokens when commiting from command line.*
- Provided by GitHub [[https://github.com/settings/tokens][here]].
#+BEGIN_SRC
- generate a token for repository changes.
- save it locally to your system, it will not be accessible again.
- enter the token when prompted for your password during a push.
#+END_SRC
* Cloning
To get your configurations (or these) on a fresh system, you simply clone the bare repository and checkout and they're automatically installed!
** Simple Clone
#+BEGIN_SRC
git clone --bare https://github.com/noncog/.dotfiles $HOME/.dotfiles
  
# alias only in this shell instance
# assumes you back up your .bashrc with the permanent alias,
# which will be installed momentarily.
  
alias dotfiles='/usr/bin/git --git-dir=$HOME/.dotfiles/ --work-tree=$HOME'

dotfiles checkout
#+END_SRC
If you have configs already in place, the above will likely fail, and ask you to remove the config files before dotfiles can be cloned in.

If this is the case, you can manually solve conflicts or use the following automatic cloning script:
- *WARNING*: If used as is, this will backup your dotfiles and replace them with mine. Change the repository for your own use.
** Automated Cloning Script
#+BEGIN_SRC
# clone dotfiles
git clone --bare https://github.com/noncog/.dotfiles $HOME/.dotfiles

# checkout will backup dotfiles in the way
cd ${HOME}
mkdir -p .dotfiles-backup && \
/usr/bin/git --git-dir=$HOME/.dotfiles/ --work-tree=$HOME checkout 2>&1 | egrep "\s+\." | awk {'print $1'} | \
xargs -I{} mv {} .dotfiles-backup/{}

# now check out
/usr/bin/git --git-dir=$HOME/.dotfiles/ --work-tree=$HOME checkout

# hide untracked files
/usr/bin/git --git-dir=$HOME/.dotfiles/ --work-tree=$HOME config --local status.showUntrackedFiles no
#+END_SRC
*** Usage
#+BEGIN_SRC
# Save this script to a file ending in .sh (dotfiles.sh)
chmod +x dotfiles.sh
./dotfiles.sh 
#+END_SRC
Again, this script is part of my [[https://github.com/noncog/debian-autosetup][debian-autosetup]] script, which may be easier for you to use to setup your software and clone dotfiles managed with a git --bare repository.
