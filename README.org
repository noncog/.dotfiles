* Overview
This system follows my philosophy of how a system should be built, maintained, and interacted with.

#+HTML: <details><summary><b>Philosophy</b></summary><p align="center"><code>Everyone has one on Linux ... these are mine.</code></p><ul><li><details><summary><b>Stability</b></summary><p><ins>I don't need or use the latest and greatest.</ins><ul><li>I would rather it just work.</li><li>Often, older tools are better supported and more powerful than "the next big thing."</li><li>I use Debian Stable to get long term support instead of something like a "rolling release" distribution.<ul><li>Run like the wind, Bullseye!</li></ul></li></ul></p></details></li><li><details><summary><b>Minimalism</b></summary><p><ins>I try to do the most with what I've got.</ins><ul><li>I prefer to use the tools I already have to accomplish what another tool could.</li><li>Sometimes I break this convention for the sake of aesthetics. e.g. <code>rofi > dmenu</code></li><li>I use a minimal install of Debian and build my entire system from a TTY to a full desktop environment.</li><li>I avoid excess. For example, I have no desktop manager, only a window manager. I am the only user of the system and anything more would be bloat.</li></ul></p></details></li><li><details><summary><b>Customizability</b></summary><p><ins>All software I use must be highly configurable and not force the user into limiting conventions.</ins><ul><li>Colors must be configurable.</li><li>Keybinds must be configurable.</li></ul><br>Thankfully, most popular software for Linux easily meets these needs.</p></details></li><li><details><summary><b>Automation</b></summary><p><ins>I don't want to waste time building a system again.</ins><ul><li>I automate the installation and versioning of my system software, setup, and dotfiles using my <a href="https://github.com/noncog/debian-autosetup">debian-autosetup</a> script.</li><li>My configurations are version controlled and managed through GitHub using a <code>git -bare repository.</code></li><li>This allows my configuration files to be easily versioned from and automatically installed to their appropriate location, just by cloning this repository! #+HTML: <h1 align="center">System</h1>
#+HTML: <p align="center"><code>&#91 <a href="#install">Install</a> - <a href="#foo">foo</a> - <a href="#git--bare-guide">Git -bare Guide</a> &#93</code></p>

It's actually very simple. The software I use and their configurations are installed by [[https://github.com/noncog/debian-autosetup][debian-autosetup]].

If you want to use my exact system the way it is, just run debian-autosetup.

You may not want to use my system, because it's custom to me. But you can still use my tools and techniques to save yourself a lot of work.

The configurations are managed in a =git -bare= repository. And debian-autosetup can install softtware and dotfiles managed with git -bare repositories.

Therefore, to have a fully version controlled system with an automatic installer, all you must do is fork debian-autosetup, customize it with your software, manage your configurations with a git -bare repository and point debian-autosetup to it.
