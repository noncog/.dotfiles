* Overview
This system follows my philosophy of how a system should be built, maintained, and interacted with.

#+HTML: <details><summary><b>Philosophy</b></summary><p align="center"><code>Everyone has one on Linux ... these are mine.</code></p><ul><li><details><summary><b>Stability</b></summary><p><ins>I don't need or use the latest and greatest.</ins><ul><li>I would rather it just work.</li><li>Often, older tools are better supported and more powerful than "the next big thing."</li></ul></p></details></li><li><details><summary><b>Minimalism</b></summary><p><ins>I try to do the most with what I've got.</ins><ul><li>I prefer to use the tools I already have to accomplish what another tool could.</li><li>Sometimes I break this convention for the sake of aesthetics. e.g. <code>rofi > dmenu</code></li></ul></p></details></li><li><details><summary><b>Customizability</b></summary><p><ins>All software I use must be highly configurable and not force the user into limiting conventions.</ins><ul><li>Colors must be configurable.</li><li>Keybinds must be configurable.</li></ul><br>Thankfully, most popular software for Linux easily meets these needs.</p></details></li><li><details><summary><b>Automation</b></summary><p><ins>I don't want to waste time building a system again.</ins><ul><li>My configurations are version controlled and managed through GitHub using a <code>git -bare repository.</code></li><li>This allows my configuration files to be installed automatically in the appropriate location by just cloning this repository. This means <ins>no copying or symlinks</ins>.</li></ul><br>To setup your own dotfiles managed with git -bare see my tutorial:</p></details></li></ul></details>
